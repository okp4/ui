# Typography

> OKP4 typography component.

[![stability-unstable](https://img.shields.io/badge/stability-unstable-yellow.svg)](https://github.com/emersion/stability-badges#unstable)

## Overview

**Typography** is the primitive component to display some text or pure HTML. It comes with limited styling capabilities to be easily integrated with OKP4 design system.

Any possible combination of the following properties can be used and are detailed below.

## Fonts

The Typography component can be styled with two font families:

### Main font

The main font is `Gotham`, it is also the default one.

## Colors

There are different color variants available referring to OKP4 palette and themes.  
These variations have been developed to allow the Typography component to fit perfectly into any interface, regardless of the theme selected.

You can choose from one of 6 basic color variants that meet most of your needs:

- **text**: the default text color, follows the selected theme.
- **highlighted-text**: the text is highlighted by a stronger color.
- **info**: a light grey text color to give some information to the user.
- **success**: a green text color to inform the user that something has succeeded.
- **warning**: an orange text color to make the user pay attention.
- **error**: a bright red text color to alert the user that an error has occurred.

To allow the management of low contrast issues or cases where the color must remain the same according to
the theme, the OKP4 palette addresses these issues by adding 2 more complex color effects:

- **inverted-text**: inverts the theme of the text color. You can use this color when the text is displayed on low contrast backgrounds.
- **invariant-text**: does not depend on the theme, it is always white.

## Font weights

Depending on the need, the font weight can be modified. The values are `xlight`, `light` (default), `bold`, `black`.
Internally, this parameter will actually select a variant of the selected font.

## Text decoration

The `textDecoration` property adds context to the Typography component and thus optimizes user actions.
At the moment, the available value is:

- **underline** : underlines the text in the same color as the text itself.

More options are already under development and will make it possible to take even more advantage of this property.

<Canvas>
  <Story name="Text decoration">
    <div>
      <Typography as="div" textDecoration="underline">
        underline | OKP4 - At the core of the stakes of data valorisation
      </Typography>
    </div>
  </Story>
</Canvas>

## Text-wrap

By default, the element rendered will automatically wrap : newline characters in the source are handled the same as other white space. Lines are broken as necessary to fill line boxes.
If `true`, the text will not wrap, but instead will truncate with a text overflow ellipsis.

<Canvas>
  <Story name="Text wrap">
    {args => (
      <div>
        <Typography as="div" fontWeight="light" fontFamily="brand" fontSize="medium" noWrap={true}>
          At the core of the stakes of data valorisation. OKP4 offers an integrated solution that is
          perfectly adapted to your use and your needs.
        </Typography>
      </div>
    )}
  </Story>
</Canvas>
