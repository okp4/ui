# Typography

> ØKP4 typography component.

[![stability-unstable](https://img.shields.io/badge/stability-unstable-yellow.svg)](https://github.com/emersion/stability-badges#unstable)

## Overview

**Typography** is the primitive component to display some text or pure HTML. It comes with limited styling capabilities to be easily integrated with OKP4 design system.

Any possible combination of the following properties can be used and are detailed below.

## Fonts

The Typography component can be styled with two font families:

### Main font

The main font is `Gotham`, it is also the default one.

<Canvas>
  <Story name="Main font">
    {args => (
      <Typography as="div" fontFamily="brand" fontSize="medium">
        <p>ABCDEFGHIJKLMNOPQRSTUVWXYZ</p>
        <p>abcdefghijklmnopqrstuvwxyz</p>
        <p>123456789 !?.;:</p>
      </Typography>
    )}
  </Story>
</Canvas>

### Secondary font

You can also choose to use a secondary font called `NotoSansMono`.

<Canvas>
  <Story name="Secondary font">
    {args => (
      <Typography as="div" fontFamily="secondary" fontSize="medium">
        <p>ABCDEFGHIJKLMNOPQRSTUVWXYZ</p>
        <p>abcdefghijklmnopqrstuvwxyz</p>
        <p>123456789 !?.;:</p>
      </Typography>
    )}
  </Story>
</Canvas>

## Sizes

Size options are `x-small`, `small`, `medium`, `large`, `x-large`.  
There is no default size to preserve what's being ultimately rendered.

## Font weights

Depending on the need, the font weight can be modified. The values are `xlight`, `light` (default), `bold`, `black`.
Internally, this parameter will actually select a variant of the selected font.

<Canvas>
  <Story name="Font weight">
    {args => (

        <Typography as="div" fontWeight="xlight" fontFamily="brand" fontSize="medium">
          xlight - OKP4 - At the core of the stakes of data valorisation
        </Typography>
        <Typography as="div" fontWeight="light" fontFamily="brand" fontSize="medium">
          light - OKP4 - At the core of the stakes of data valorisation

## Text-wrap

By default, the element rendered will automatically wrap : newline characters in the source are handled the same as other white space. Lines are broken as necessary to fill line boxes.
If `true`, the text will not wrap, but instead will truncate with a text overflow ellipsis.

<Canvas>
  <Story name="Text wrap">
    {args => (
      <div>
        <Typography as="div" fontWeight="light" fontFamily="brand" fontSize="medium" noWrap={true}>
          At the core of the stakes of data valorisation. OKP4 offers an integrated solution that is
          perfectly adapted to your use and your needs.
        </Typography>
      </div>
    )}
  </Story>
</Canvas>
